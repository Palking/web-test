Top-Down-Tiles

The idea behind the game was to create a small, 2D game using C# with the MonoDevelop framework.
It was supposed to be a simple RPG with only core features like levelups and weapons implemented.

I stopped working on it, because i was unhappy with my solutions for collision detection and started to like Unity a lot more.

The main features i had implemented where the movement (8 direction based), firing some projectiles as well as collision with obstacles constructed by using a tile based map, that is generated by using different tile types and an array to contain the level information.

myCookbook

My main 'non-game'-project.

It is currently the project that i spent the most time working on and is still far from finished.
The finished solution is supposed to be a website for sharing and storing recipes.
This will make use of my raspberry pi to host both the data containing the information (currently in plain .txt files, will probably suffice) as well as hosting the web API including the whole backend logic like input checking and so on.

The backend is written in C#, the website will be either in .NET's MVC or, more likely, with the pure HTML/CSS/JS/jquery stack using requests to the API.

When all of those basic functions are working, i plan to also make a android application for a more convinient experience as well as a dashboard using my pi for the kitchen.
